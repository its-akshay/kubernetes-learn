Cluster       A group of machines managed as one unit
Node          A single machine (VM/physical) in the cluster
Pod           Smallest deployable unit — wraps 1+ containers
Deployment    Declares desired state: '3 replicas of my Go server'
Service       Stable network endpoint in front of a set of Pods
Namespace     Virtual cluster for isolating teams or environments
kubectl       CLI tool to interact with the Kubernetes API
YAML          manifest File that describes what you want K8s to create/maintain


===========2=========
Control Plane (Master)
• kube-apiserver — REST API front door; every kubectl command hits this.
• etcd — distributed key-value store, single source of truth for cluster state.
• kube-scheduler — picks which node a new Pod should run on.
• kube-controller-manager — runs controllers (ReplicaSet, Node, Endpoint...).
Worker Nodes
• kubelet — agent on every node; starts/stops containers as instructed.
• kube-proxy — maintains iptables rules for Service routing.
• Container Runtime — actually runs containers (containerd, CRI-O).
